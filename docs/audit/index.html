Turbolong Audit Summary

This page repurposes the Blend Protocol security research into a Turbolong-specific, user-facing summary. It reflects the current audit state as of commit cebe5a58b273394552a00fa02f6388058db3b4fa.

Key findings

  • Same-asset leverage loops on Blend can temporarily dilute utilization and bypass the configured max_util cap.
  • This attack path enables large withdrawals and forces borrow APRs into the steep penalty curve, which can quickly degrade borrower health factors.
  • On Stellar mainnet, the Etherfuse USDC reserve can reach >95% utilization, where the Blend rate model can spike borrow APRs above 500%.

What changed since the audit

The current repository state includes the public vulnerability report and supporting tooling. As of the audited commit, the issue is documented and remains acknowledged in the report; mitigation work should be tracked separately from this public summary.

Outstanding issues acknowledged here include:

  • Blend contract position and utilization checks remain subject to the same-asset loop risk.
  • Pool-level withdrawal checks still require stronger enforcement of max_util during withdraw operations.
  • Oracles and high-leverage asset config should be hardened to limit implicitly amplified rate and liquidation risk.
